# CubeRhythmEditor 中文文档

## 项目简介

CubeRhythmEditor 是一个为 CubeRhythm（Minecraft 节奏游戏插件）设计的谱面编辑器。使用 Java 17 + Swing 构建，支持在 7×7 网格上放置各种类型的音符，并与 OGG 音频文件同步播放，最终导出为 JSON 格式的谱面文件。

---

## 使用方法

### 启动

```bash
# 方式一：Maven 直接运行
mvn exec:java

# 方式二：打包后运行
mvn clean package
java -jar target/CubeRhythmEditor-1.0.0-SNAPSHOT.jar
```

### 界面布局

启动后窗口分为以下区域：

- **左侧**：网格编辑区（Grid / No-Grid 两种模式）
- **右上**：音符类型选择面板（彩色单选按钮）
- **右中**：歌曲信息面板（可折叠）、音符属性面板（方向/发光）
- **右下**：谱面 JSON 编辑区 + 操作日志
- **底部**：音乐播放控制条（进度条、时间显示、小节/拍号）

### 基本工作流程

1. **导入音乐**：菜单 → 导入 OGG 音频文件
2. **设置歌曲信息**：填写歌名、作曲、谱师、BPM、难度等
3. **选择音符类型**：在右侧面板选择 TAP/DRAG/HOLD/DOUBLE/FLICK/EXECUTION
4. **放置音符**：在网格上点击放置，音符会绑定到当前播放时间
5. **调整属性**：设置方向（W/A/S/D）和发光效果
6. **预览**：播放音乐，观察伪 3D 动画效果
7. **导出**：保存为 JSON 谱面文件

## 编辑模式

### Grid 模式（网格模式）

- 坐标自动吸附到 0.5 单位的网格点
- 适合快速精确放置
- 点击网格交叉点即可放置音符

### No-Grid 模式（自由模式）

- 允许任意精度的坐标（保留两位小数）
- 两种输入方式：
  - **鼠标跟随模式**：鼠标移动时显示跟随方块，点击放置
  - **手动输入模式**：在坐标输入框中输入精确坐标，点击确认放置

切换方式：点击模式切换按钮

## 音符类型详解

### TAP（点击）
- 颜色：蓝色
- 最基础的音符，玩家需要在正确时间点击对应位置
- 属性：坐标 (x, y)、方向、发光

### DRAG（滑动）
- 颜色：黄色
- 玩家需要向指定方向滑动
- 属性：坐标 (x, y)、方向、发光

### HOLD（长按）
- 颜色：绿色
- 玩家需要按住不放
- 属性：坐标 (x, y)、方向、发光

### DOUBLE（双押）
- 颜色：橙色
- 需要同时点击两个位置
- 放置方式：先点击第一个位置，再点击第二个位置完成放置
- 放置过程中不能切换音符类型
- 属性：两组坐标 (x1, y1) 和 (x2, y2)、方向、发光

### FLICK_LEFT / FLICK_RIGHT（左滑/右滑）
- 颜色：红色（左）/ 紫色（右）
- 全屏方向性音符，不需要坐标
- 属性：方向、发光、turn（left/right）

### EXECUTION（执行事件）
- 颜色：灰色
- 不是游戏音符，而是在指定时间触发 Minecraft 事件
- 无坐标、无方向、无发光
- 支持的 action 类型：
  - `title`：显示标题（支持 Minecraft 格式码 §）
  - `actionbar`：显示动作栏文字
  - `chat`：发送聊天消息
  - `potion`：施加药水效果
  - `clear_effects`：清除所有效果

## 方向系统

每个音符（EXECUTION 和 FLICK 除外）都有一个方向属性，对应键盘 WASD：

| 方向 | 含义 | 边框颜色 |
|----|----|------|
| W  | 上  | 白色   |
| A  | 左  | 黄色   |
| S  | 下  | 橙色   |
| D  | 右  | 红色   |

方向决定了音符在编辑器中的边框颜色，也决定了玩家需要面朝的方向。

## 音乐播放控制

- **播放/暂停**：控制音乐播放
- **进度条**：拖动跳转到指定位置
- **拍号吸附**：自动对齐到最近的拍子
- **键盘快捷键**：左右方向键按拍跳转
- **Offset 偏移**：调整音乐与谱面的同步偏移量

## 伪 3D 渲染系统

这是编辑器最有特色的实现之一。音符不是简单地平铺显示，而是模拟了一个从远处飞来的 3D 效果：

### 实现原理

1. **显示范围**：由 `displayBeatsCount`（默认 8 拍）控制可见音符的时间范围
2. **进度插值**：每个音符根据其时间与当前播放时间的差值，计算一个 0.0~1.0 的进度值
3. **缩放**：进度越接近 1.0（即将到达），音符越大；越接近 0.0（远处），音符越小
4. **透明度**：远处的音符更透明，近处的更不透明（Alpha 混合）
5. **容差**：使用 0.1 拍的容差确保音符在边界处不会突然消失

### 渲染细节

```
进度 = 1.0 - (音符时间 - 当前时间) / 显示范围
缩放比例 = 进度 × 最大尺寸
透明度 = 进度 × 255
```

音符从网格中心向外扩展，模拟从远处飞向玩家的效果。

## JSON 谱面格式

### 完整结构

```json
{
  "version": "1.0.0",
  "metadata": {
    "id": "唯一标识",
    "title": "歌曲名",
    "artist": "作曲",
    "charter": "谱师",
    "difficulty": {
      "name": "Easy/Normal/Hard",
      "level": 1-15,
      "color": "AQUA"
    },
    "audio": "cr.chart_id",
    "duration": 60,
    "offset": 0,
    "bpm": 120
  },
  "notes": []
}
```

### 各音符类型的 JSON 表示

**TAP / DRAG / HOLD：**
```json
{
  "type": "tap",
  "time": 5.0,
  "face": "w",
  "position": {"x": 0, "y": 0},
  "glowing": false,
  "tag": ""
}
```

**DOUBLE：**
```json
{
  "type": "double",
  "time": 15.0,
  "face": "w",
  "positions": [
    {"x": -1, "y": -1},
    {"x": 1, "y": 1}
  ],
  "glowing": false,
  "tag": ""
}
```

**FLICK：**
```json
{
  "type": "flick",
  "time": 12.0,
  "face": "w",
  "turn": "left",
  "glowing": false,
  "tag": ""
}
```

**EXECUTION：**
```json
{
  "type": "execution",
  "time": 2.0,
  "actions": [
    {
      "type": "title",
      "enabled": true,
      "title": "§6§lCubeRhythm",
      "subtitle": "§e准备开始！",
      "fadeIn": 10,
      "stay": 40,
      "fadeOut": 10
    }
  ]
}
```

### 时间单位

- JSON 中的 `time` 字段单位为**秒**（浮点数，如 `5.0` 表示第 5 秒）
- 编辑器内部使用**微秒**（long 类型），转换公式：`microseconds = seconds × 1,000,000`

## 特殊实现细节

### JSON 解析方式

编辑器的 JSON 导入并未使用 Gson 库进行反序列化，而是采用**正则表达式手动解析**。这是因为谱面 JSON 结构相对固定，且需要处理一些特殊情况（如 DOUBLE 音符的嵌套数组）。

关键解析方法（位于 `MainWindow.java`）：
- `extractJsonString(json, key)` — 提取字符串值
- `extractJsonNumber(json, key)` — 提取数字值
- `extractJsonBoolean(json, key)` — 提取布尔值
- `extractJsonObject(json, key)` — 提取嵌套对象
- `extractJsonArray(json, key)` — 提取数组

### NoteManager 的 TreeMap 设计

使用 `TreeMap<Long, List<Note>>` 存储音符，key 为微秒时间戳。这样设计的优势：
- 音符按时间自动排序
- 可以高效地进行范围查询（`subMap`），只获取当前可见时间范围内的音符
- 同一时间点可以有多个音符（List）

### 音频解码流程

1. 使用 j-ogg-all 库读取 OGG 文件
2. 解码为 PCM_SIGNED 16-bit 格式
3. 通过 Java Sound API 的 Clip 接口播放
4. 支持 seek（跳转）、pause（暂停）、获取当前播放位置

### UI 刷新机制

- 使用 Swing Timer（50ms 间隔）定时刷新 UI
- 每次刷新时从 Clip 获取当前播放位置
- 更新进度条、时间标签、小节/拍号显示
- 触发网格面板重绘以更新音符动画

### Minecraft 格式码

EXECUTION 音符的文本字段支持 Minecraft 格式码（§ 前缀）：
- `§0-§9, §a-§f`：颜色代码
- `§l`：粗体
- `§o`：斜体
- `§n`：下划线
- `§m`：删除线
- `§r`：重置格式

示例：`§6§lCubeRhythm` = 金色粗体文字

## 坐标系统详解

```
        Y轴 (+3.5)
          ↑
          |
(-3.5) ←--+--→ (+3.5) X轴
          |
          ↓
        Y轴 (-3.5)
```

- 原点 (0, 0) 位于网格中心
- 每个格子代表 1 个单位
- Grid 模式下坐标吸附到 0.5 的倍数（如 -1.5, -1.0, -0.5, 0, 0.5...）
- No-Grid 模式下可以放置到任意位置（精度 0.01）

---

## 菜单功能

- **导入音乐**：选择 OGG 文件加载
- **导入谱面**：从 JSON 文件加载谱面（可选合并或覆盖）
- **保存谱面**：将当前谱面导出为 JSON 文件
- **显示拍数**：调整伪 3D 动画的可见范围（4/8/16/32 拍）
- **Grid/No-Grid 切换**：切换编辑模式
