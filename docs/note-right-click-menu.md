# 音符右键菜单设计

## 背景

当前用户无法直接在谱面视图中修改已放置的音符，只能手动在 JSON 编辑区查找并修改，体验很差。

## 目标功能

右键点击谱面视图中的音符时，弹出上下文菜单，提供以下操作：

### 菜单项（初步设想）

1. **删除音符** — 立即删除该音符（当前左键已有此功能，右键菜单提供更明确的入口）
2. **定位到 JSON** — 将 JSON 编辑区的光标和视图滚动到该音符对应的 JSON 片段处
3. **修改方向（face）** — 子菜单：W / A / S / D
4. **切换 glow** — 勾选项，切换 `glowing` 字段
5. **修改音符类型** — 子菜单：TAP / DRAG / HOLD / DOUBLE / FLICK_LEFT / FLICK_RIGHT / EXECUTION（类型转换有约束，见下）

## 技术要点

### 音符命中检测

`findNoteAtPosition(mouseX, mouseY, panel)` 已在 `MainWindow.java:3000` 实现，可直接复用。当前它被用于左键删除逻辑，需要确认它在无网格和网格两种模式下都能准确命中。

**待确认：** `findNoteAtPosition` 的命中范围（像素容差）是多少？是否需要调整以适配右键场景？

**用户回答** 我们只判定在面上的音符；光标在音符内即判定有效，并展示右键菜单

### "定位到 JSON" 实现思路

`JsonEditorPanel` 使用 `RSyntaxTextArea`，支持 `textArea.setCaretPosition(int)` 和 `textArea.scrollRectToVisible(...)`。

需要在 JSON 文本中搜索该音符的特征字段（如 `"time": <seconds>`）来定位偏移量。由于同一时刻可能有多个音符，需要进一步用坐标或类型区分。

**待确认：** 同一时刻若有多个音符（如 DOUBLE），定位逻辑如何选择正确的那个？

**用户回答** 先主动找寻配置文件中完全符合参数(是否发光，所在屏幕位置等)的配置项，如果有重复的，指向第一个即可

### 类型转换约束

不同类型的音符结构差异大（DOUBLE 有两个坐标，EXECUTION 有 actions 数组，FLICK 没有坐标），随意切换类型会导致字段缺失。

**待确认：** 是否要支持类型切换？如果支持，缺失字段填默认值还是弹对话框让用户填写？

**用户回答** 不支持，限制多，操作复杂

### 鼠标事件

当前 `mouseClicked` 里左键 = 放置/删除，右键 = 无操作（或未处理）。需要在右键 `mouseClicked` 或 `mousePressed` 里调用 `findNoteAtPosition`，命中则弹菜单，未命中则不弹。

**待确认：** 右键未命中音符时是否需要弹出"在此处放置…"之类的菜单，或完全不处理？

**用户回答** 要弹出带有提示词的空菜单，顺带一提，目前的测试中发现删除功能绑定在右键，或许是程序没有指明，但实际上右键才是删除音符的方式，左键只能放置音符

### 用户补充

当弹出右键菜单时，点击屏幕任意位置关闭菜单，若左键主面板内，关闭右键菜单但是不放置音符(防误触处理)
